Managing User-Defined Functions

      +
      User-defined functions can be managed through the Query Workbench

      The Query WorkBench can be used to create your own user-defined functions as part of your Capella instance. If you are unfamiliar with the workbench, then you will find more information here: Accessing the Query Workbench

      Creating a User-Defined Function

      Use the Query Editor to execute a SQL++ call that creates your function.

      Listing 1. Function to convert feet to meters.
      CREATE FUNCTION to_meters(...) { args[0] * 0.3048 }

      The statement can be viewed in three parts:

      CREATE FUNCTION

      The SQL++ command that will define the function.
      to_meters(…​)

      The name and parameter definition list of the function. Instead of using a list of named parameters, the function uses a variadic parameter list (denoted by the three dots) which means that a variable number of parameters can be supplied to the function and will translate these into a list of values.
      { args[0] * 0.3048 }

      The body of the function is delimited by brackets. The arguments received are a list of values referenced by a zero-based index (so in this case, the first value in the list is referenced by args[0]).
      Named Parameters

      Named parameters are also supported. As an example, the equivalent to_meters function using named parameters would look like this:

      CREATE FUNCTION to_meters(feet) { feet * 0.3048 }

      Execute this function in the Query Editor:

      Figure 1. Create User-Defined Function in Query Editor

      Once the function has been created, it can be used as part of a SQL++ statement which returns a list of airports along with their mean above seal level expressed in meters.

      SELECT airportname, ROUND(to_meters(geo.alt)) AS mamsl
FROM `travel-sample`.inventory.airport
LIMIT 5;
      Figure 2. Executing a UDF from the Query Editor

      You can find more information on CREATE FUNCTION here: SQL++ CREATE FUNCTION

      View User-Defined Functions

      You can view the user-defined functions on your Capella instance by executing the following command in the Query Editor:

      Listing 2. Statement to list user-defined functions
      SELECT * FROM system:functions
      Listing 3. Result from selecting system functions
      [
  {
    "functions": {
      "definition": {
        "#language": "inline",
        "expression": "((`args`[0]) * 0.3048)",
        "text": "args[0] * 0.3048"
      },
      "identity": {
        "name": "to_meters",
        "namespace": "default", (1)
        "type": "global" (2)
      }
    }
  }
]
      1 The function resides in the default namespace.
      2 The default namespace is globally accessible.

      Remove a User-Defined Function

      Execute the DROP FUNCTION statement in the Query Editor to remove a user-defined function from a Capella instance.

      DROP FUNCTION to_meters

      Run the SELECT * FROM system:functions statement to ensure that the function has been removed.

      You cannot change a function once it has been created. If you need to change a function, DROP it and then CREATE it again with the changes.

      Function Scopes

      Up until this point, the functions created here have been defined in global namespace as (demonstrated in Listing 3).

      You can also create functions that reside within scope.

      The travel-sample bucket contains a number of scopes, one of which is called inventory.

      To create the function to_meters as part of the inventory scope, use the following statement:

      CREATE FUNCTION default:`travel-sample`.inventory.to_meters(...) { (args[0] * 0.3048) }

      Now that the function is defined as part of a scope, you must use the complete scope reference to call it:

      SELECT airportname, ROUND(default:`travel-sample`.inventory.to_meters(geo.alt)) AS mamsl
FROM `travel-sample`.inventory.airport
LIMIT 5;

      For more information on scopes, see Manage Scopes and Collections
